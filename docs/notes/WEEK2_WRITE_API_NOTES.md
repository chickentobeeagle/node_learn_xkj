# Week 2 写入接口说明（POST /posts 与 POST /comments）

## 1) 为什么写入前要先校验

### 创建文章
- 先校验 `userId/title/content` 是否存在且格式正确
- 再校验 `userId` 对应用户是否存在

原因：  
如果直接写库，外键错误会变成数据库报错信息，不利于前端理解。  
先在接口层校验，可以返回更明确的 400/404。

### 创建评论
- 先校验 `postId/userId/content`
- 再确认文章存在
- 再确认评论用户存在

原因：  
评论是“文章 + 用户”双外键关系，任一不存在都不该写入。

## 2) 状态码语义

- `400` 参数不合法（缺字段、格式错误）
- `404` 关联资源不存在（文章不存在、用户不存在）
- `201` 创建成功
- `500` 未预期的服务端错误

## 3) 为什么加 `GET /posts/:id/comments`

加这条查询接口是为了形成完整闭环：

1. `POST /posts` 创建文章  
2. `POST /comments` 创建评论  
3. `GET /posts/:id/comments` 立刻验证写入是否成功

这样学习路径从“只会查”升级为“可写可查可验证”。
