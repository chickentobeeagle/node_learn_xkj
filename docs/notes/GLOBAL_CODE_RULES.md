# 全局代码规则（学习项目）

## 1. 注释规则（强制）
- 所有**变量定义**必须有注释说明其业务含义（不是只解释类型）。
- 所有**函数/方法定义**必须有注释说明：
  - 输入参数含义
  - 核心处理逻辑
  - 返回结果语义（尤其是状态码和错误分支）
- 所有**HTTP 接口路由**必须有注释说明：
  - 路径与请求方法（如 `GET /users/:id`）
  - 参数来源（`params/query/body`）
  - 成功与失败的主要返回语义
- 注释统一使用**中文**，优先写“为什么这样设计”，不只写“代码做了什么”。

## 2. 分层职责规则（强制）
- `routes` 层：只负责接收请求、调用 service、返回响应。
- `services` 层：负责参数校验、业务编排、状态码语义。
- `repos` 层：只负责 SQL 执行与数据读写，不做 HTTP 语义判断。
- `db` 层：负责连接管理、基础 SQL 封装、schema 初始化。

## 3. 响应协议规则（强制）
- 所有接口统一返回结构：`code/message/data`。
- 状态码语义保持一致：
  - `200/201` 成功
  - `400` 参数错误
  - `404` 资源不存在
  - `409` 资源冲突（如 email 重复）
  - `500` 服务内部错误

## 4. 安全与可维护性规则（强制）
- SQL 条件值必须使用参数占位符，禁止字符串拼接值。
- `ORDER BY` 字段必须走白名单，不允许直接信任用户输入。
- 分页参数必须做默认值与上限保护，避免大分页拖垮接口。
